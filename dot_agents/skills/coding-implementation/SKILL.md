# コード実装スキル

## コメントの記述

* 大まかな処理の単位で1行の短いコメントを書く
* すべてのメソッドにドキュメントコメントを書く
* グローバル変数、クラス変数、インスタンス変数などの寿命が長い変数にはドキュメントコメントを書く
* コメントは日本語で書く

## 設計原則

* 単一責任の原則に従う
* YAGNI原則に従う
* オープン・クローズドの原則に従う
* デメテルの法則に従う
* リスコフの置換原則に従う

## 実装チェック項目

* メモリリークしていないか確認する
* セキュリティの観点から実装を見直す（CIA：機密性、完全性、可用性）
    * 機密性（Confidentiality）：許可されていない人に情報が見られないようにする
    * 完全性（Integrity）：情報が改ざんされないようにする
    * 可用性（Availability）：必要な時にシステムが利用可能であるようにする

## 命名ルール

* クラス名、変数名、メソッド名などは品詞に注意して命名する
    * メソッドの呼び出し元で自然な英語になっていることが理想
* 変数名に否定形は使わない

## アーキテクチャ

* 必要に応じて依存性の注入、依存性逆転の原則を活用する
    * 不必要に使わないこと
