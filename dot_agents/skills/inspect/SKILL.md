# コード・ドキュメントレビュースキル

## レビューの目的

コードとドキュメントの品質を向上させるため、改善が必要な項目を重要度の高い順に1件ずつ提示し、解決を支援します。

## レビューワークフロー

1. レビュー対象を確認（ファイルパス、Globパターン、ブランチ差分など）
2. 改善項目を重要度順（Critical → Major → Minor）にリストアップ
3. 最初の項目を提示
4. ユーザーの修正・確認を待つ
5. 完了後、次の項目に進む（ステップ3へ）
6. すべての項目を完了したら終了

## 各ステップの対応

### 項目の提示時

* ファイルパスと行番号を明記
* 改善が必要な理由を簡潔に説明
* 改善方法を2～3行で提案

### 修正後の対応

* ユーザーの修正内容を確認
* 修正が適切であれば「✓ 完了」と記録
* 不十分であれば追加の修正を提案

## 重要度レベルの定義

* Critical: セキュリティ上の問題、致命的なバグ、実装エラー
* Major: アーキテクチャやデザイン上の改善、パフォーマンスの問題、保守性の低下
* Minor: コードスタイルの統一、名前付けの改善、ドキュメントの明確化

## 評価基準

* コードレビュー → coding-implementation スキルに準ずる
* ドキュメントレビュー → documentation-writing スキルに準ずる
* 修正の必要がない項目には言及しない
