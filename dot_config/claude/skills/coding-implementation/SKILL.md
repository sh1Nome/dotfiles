# コード実装スキル

## コメントの記述

* 大まかな処理の単位で1行の短いコメントを書く
* メソッドや変数には LSP 対応フォーマットのコメントを書く
* コメントは日本語で書く

## 実装時の方針

* 単一責任の原則に従う
* YAGNI原則に従う
* オープン・クローズドの原則に従う
* デメテルの法則に従う
* リスコフの置換原則に従う

## 命名ルール

* クラス名、変数名、メソッド名などは品詞に注意して命名する
    * メソッドの呼び出し元で自然な英語になっていることが理想
* 変数名に否定形は使わない

## アーキテクチャ

* 必要に応じて依存性の注入、依存性逆転の原則を活用する
    * 不必要に使わないこと
