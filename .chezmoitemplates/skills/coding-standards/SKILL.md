# Coding Standards

## ワークフロー

1. 設計を考える（設計原則を意識する）
2. 実装する（命名規則・コメント方針に従う）
3. チェックリストで検証する

## チェックリスト

### 設計原則

- [ ] 単一責任の原則に従う設計か
- [ ] YAGNI原則（不要な機能を実装していないか）を確認
- [ ] 求められていないフォールバック処理を追加していないか
- [ ] オープン・クローズドの原則を適用できるか
- [ ] デメテルの法則に従い、依存性を管理しているか
- [ ] 依存性の注入・依存性逆転の原則を適切に使用しているか
- [ ] リスコフの置換原則を守れているか

### コメント

- [ ] コードを読めば分かることをコメントで繰り返していないか
- [ ] 採用しなかった実装や自明でない制約を Why not としてコメントに残しているか

### 命名規則

- [ ] クラス名・メソッド名・変数名は意図が明確か
- [ ] 呼び出し元で自然な文になっているか
- [ ] 否定形を避けられているか
- [ ] 品詞に注意した命名か（メソッドは動詞で始まるなど）

### 品質チェック

- [ ] メモリリークの可能性を確認したか
- [ ] セキュリティを確認（CIA）：
  - 機密性（Confidentiality）：許可されたユーザーのみアクセス可能か
  - 完全性（Integrity）：データが改ざんされないか
  - 可用性（Availability）：システムが利用可能か
- [ ] パフォーマンスに問題はないか
