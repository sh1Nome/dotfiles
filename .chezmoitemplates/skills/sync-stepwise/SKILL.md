# sync-stepwise

## Quick start

```
> この機能について認識を合わせたい

質問:
この機能で解決したい問題は何ですか？

推奨:
パフォーマンスのボトルネックを解消したい
```

ユーザーの回答を受けたら次の質問へ進む。

## ワークフロー

1. ユーザーのテーマ・文脈を把握する
2. 決定事項を木構造として捉え、各分岐と依存関係を整理する
3. 依存関係を考慮した順に、問いを1件提示する
4. すべての分岐を辿り終えるまでステップ3を繰り返す

## 質問の提示形式

```
質問:
質問文

推奨:
推奨文
```

質問も推奨も複数行にわたってよい。推奨は断言ではなく提案として提示する。

## 徹底的に質問するための指針

* ユーザーの回答が曖昧なら、具体例を求めるか言い換えて再確認する
* 前の回答と矛盾する内容が出たら指摘して解消する
* 「なんとなく」「たぶん」「とりあえず」は掘り下げるサイン
* 一見完結しても、未確認の分岐がなくなるまで質問をやめない

## 終了条件

ユーザーから明示的な承認（「OK」「それでいい」「進めて」など）を得るまで、次のステップへ進んではならない。承認がない場合は質問を続けるか、ユーザーの指示を待つ。
