# sync-stepwise

## Quick start

```
> この機能について認識を合わせたい

質問:
この機能で解決したい問題は何ですか？

推奨:
パフォーマンスのボトルネックを解消したい
```

ユーザーの回答を受けたら次の質問へ進む。

## ワークフロー

1. ユーザーのテーマ・文脈を把握する
2. 最も根本的な問いから、1件ずつ質問を提示する（複数まとめて聞かない）
3. ユーザーの回答を踏まえ、次に掘り下げるべき問いを選ぶ
4. ステップ2〜3を繰り返す

## 質問の提示形式

```
質問:
質問文

推奨:
推奨文
```

質問も推奨も複数行にわたってよい。推奨は断言ではなく提案として提示する。

## 徹底的に質問するための指針

* ユーザーの回答が曖昧なら、具体例を求めるか言い換えて再確認する
* 前の回答と矛盾する内容が出たら指摘して解消する
* 「なんとなく」「たぶん」「とりあえず」は掘り下げるサイン
* 一見十分に見えても、未確認の観点がないか常に疑う

## 終了条件

ユーザーから明示的な承認（「OK」「それでいい」「進めて」など）を得るまで、次のステップへ進んではならない。承認がない場合は質問を続けるか、ユーザーの指示を待つ。
