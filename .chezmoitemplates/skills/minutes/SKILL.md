# minutes

## ワークフロー

1. 入力の箇条書きテキストを解析する
2. 日時・参加者をテキストから抽出する
3. どちらか一方でもあれば開催情報セクションを出力する、両方なければ省略する
4. 入力の構成にとらわれず、議題ごとの見出しを決める
5. 各見出しの内容を文章でまとめる
6. TODOを抽出して専用セクションに集約する
7. documentation-writing スキルを使い、議事録全体の文章を書く・推敲する
8. 以下のテンプレートに従って CommonMark Markdown で出力する

改善案や明確化が必要な点があれば、議事録の前に質問を添えてもよい。

## 出力形式

完成した議事録のみを出力する。前置き・後置き・コメントは一切付けない。

```
# 議事録

## 開催情報
- 日時: YYYY-MM-DD
- 参加者: 氏名1、氏名2

## 議題A
内容

## 議題B
内容

## TODO
| #   | 内容 | 担当   | 期限  |
| --- | ---- | ------ | ----- |
| 1   | ...  | 担当者 | MM/DD |
```

## ルール

- TODO の担当・期限が不明な場合は `-` を入れる
- 見出し名は具体的・説明的にする（見出し一覧を眺めるだけで全体の内容が把握できる粒度）
- 議題セクションの本文は段落形式を基本とし、列挙が適切な箇所のみ箇条書きを使う
